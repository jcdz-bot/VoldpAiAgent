## 1. 项目简介

火山AI智能体（VoldpAiAgent）是一个基于火山服务器软件开发平台（Voldp Server Software Development Platform，简称“火山语言”）编程技术开发的服务器项目，提供功能完备的AI智能体Web界面。项目以火山服务器软件开发平台的 `.wsv` 源文件（内嵌 Go 代码）作为后端，以前端 HTML/JS 作为交互界面，能够在浏览器中完成与AI模型的对话、任务调度、文件管理、技能调用、记忆维护等操作。

主要能力包括：

- **AI对话**：支持多家模型供应商（DeepSeek、硅基流动、MiMo、自定义 OpenAI 兼容端点），可流式输出、展示思考过程、调用工具。
- **看板任务**：内置看板工作流，按 `分流 → 待办 → 就绪 → 运行中 → 已完成 / 已阻塞` 流转，并支持派发器自动认领就绪任务并启动 worker 子进程执行。
- **技能系统**：基于 `SKILL.md` 元数据组织的技能目录，支持热重载与斜杠命令调用。
- **记忆系统**：通过 `SOUL.md`、`USER.md`、`MEMORY.md` 等文件维护智能体人格、用户画像与跨会话记忆。
- **工作区与终端**：浏览器内浏览文件树、预览文件、编辑代码，并可在嵌入式终端中执行命令。
- **多Profile**：每个Profile拥有独立的配置、会话、技能与记忆，互不干扰。
- **定时任务**：支持按 cron 表达式或自然语言调度执行智能体任务或脚本。
- **MCP集成**：可配置模型上下文协议（MCP）服务器，扩展工具能力。
- **多渠道接入**：可通过 Telegram、飞书、钉钉、微信、WhatsApp 等即时通讯渠道与智能体交互。

## 2. 安装与部署

### 2.1 环境要求

- 跨平台开箱即用：提供 Windows、Linux、macOS、Android 等多平台二进制文件，下载即用，无需复杂部署。

### 2.2 首次启动

#### Windows

1. 点击VoldpAiAgent.exe可执行文件。
2. 控制台将输出如下启动信息：

```text
[webui] VoldpAiAgent 火山语言版本启动中...
[ok] 配置初始化完成, 端口: 8787
  VoldpAiAgent listening on http://127.0.0.1:8787
  Then open:     http://localhost:8787
```

#### Linux / macOS

1. 将 `VoldpAiAgent_linux_amd64` 上传到目标机器（前端静态文件已内嵌于二进制，运行时自动释放，无需额外上传 `static/` 目录）。
2. 赋权并运行：

```bash
chmod +x VoldpAiAgent_linux_amd64
./VoldpAiAgent_linux_amd64
```

macOS 将 `GOOS` 改为 `darwin`，Apple Silicon 将 `GOARCH` 改为 `arm64`。首次运行若被 Gatekeeper 拦截，在「系统设置 → 隐私与安全性」中点击「仍要打开」，或执行 `xattr -d com.apple.quarantine <可执行文件>`。

#### Android

设备需开启 USB 调试。运行后用设备浏览器访问 `http://127.0.0.1:8787`，或用 `adb forward tcp:8787 tcp:8787` 转发到电脑访问。

启动流程共分15步，依次为：启用控制台颜色 → 初始化配置 → 初始化数据库 → 检测看板 worker 模式 → 同步会话 → 会话恢复 → 初始化会话生命周期 → 加载插件 → 创建引擎 → 注册CORS中间件 → 注册认证中间件 → 注册CSRF中间件 → 注册请求体限制中间件 → 注册所有API路由 → 注册静态文件服务 → 启动平台适配器 → 启动HTTP服务器。

### 2.3 配置文件 config.yaml

主配置文件 `config.yaml` 的默认路径为 `$HERMES_HOME/config.yaml`（Windows 下为 `%LOCALAPPDATA%\hermes\config.yaml`，Linux/macOS/Android 下为 `~/.hermes/config.yaml`），可通过环境变量 `HERMES_CONFIG_PATH` 覆盖。该文件支持 `${VAR}` 形式的环境变量引用，会在运行时自动展开。

常用配置字段如下：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `host` | 监听主机 | `127.0.0.1` |
| `port` | 监听端口 | `8787` |
| `model` | 默认模型与供应商（可为对象，含 `default`、`provider`、`base_url`） | 空 |
| `context_length` | 上下文长度 | `128000` |
| `threshold_tokens` | 上下文压缩阈值 | `115200` |
| `max_upload_mb` | 上传大小上限（MB） | `50` |
| `request_timeout` | 请求超时（秒） | `300` |
| `ai_request_timeout` | AI 请求超时（秒） | `600` |
| `log_level` | 日志级别 | `info` |
| `cors_origin` | CORS 允许来源 | 空 |

### 2.4 配置合并链

VoldpAiAgent 的配置来自多个来源，按以下优先级从低到高合并（后者覆盖前者）：

```text
默认值  <  config.yaml 文件  <  数据库 settings 表  <  环境变量（HERMES_ 前缀）  <  内存覆盖
```

- **默认值**：代码内置的初始默认值（如端口 `8787`、主机 `127.0.0.1`）。
- **config.yaml 文件**：YAML 格式的主配置文件，支持环境变量展开。
- **数据库 settings 表**：通过Web界面「设置」页面保存的设置项，持久化到 `httpServer.db` 的 `settings` 表。
- **环境变量**：以 `HERMES_` 前缀的环境变量，优先级高于数据库与YAML。
- **内存覆盖**：运行时通过代码设置的内存级覆盖，优先级最高，重启后失效。

### 2.5 环境变量

常用环境变量（均以 `HERMES_` 前缀）如下表所示：

| 环境变量 | 说明 | 默认值 |
|---------|------|--------|
| `HERMES_HOME` | 主目录 | Windows: `%LOCALAPPDATA%\hermes`；其他平台: `~/.hermes` |
| `HERMES_CONFIG_PATH` | `config.yaml` 路径 | `$HERMES_HOME/config.yaml` |
| `HERMES_WEBUI_HOST` | 监听主机 | `127.0.0.1` |
| `HERMES_WEBUI_PORT` | 监听端口 | `8787` |
| `HERMES_WEBUI_PASSWORD` | Web 登录密码（哈希） | 空（无密码） |
| `HERMES_WEBUI_DEFAULT_MODEL` | 默认模型 | 空 |
| `HERMES_WEBUI_DEFAULT_PROVIDER` | 默认供应商 | 空 |
| `HERMES_WEBUI_DEFAULT_WORKSPACE` | 默认工作区目录 | 当前目录 |
| `HERMES_WEBUI_STATE_DIR` | 状态目录 | `$HERMES_HOME/webui` |
| `HERMES_WEBUI_CONTEXT_LENGTH` | 上下文长度 | `128000` |
| `HERMES_WEBUI_THRESHOLD_TOKENS` | 上下文压缩阈值 | `115200` |
| `HERMES_WEBUI_MAX_UPLOAD_MB` | 上传大小上限（MB） | `50` |
| `HERMES_WEBUI_REQUEST_TIMEOUT` | 请求超时（秒） | `300` |
| `HERMES_WEBUI_AI_REQUEST_TIMEOUT` | AI 请求超时（秒） | `600` |
| `HERMES_WEBUI_MAX_ITERATIONS` | 最大迭代次数（0=不设限，1-200） | `0` |
| `HERMES_WEBUI_CUSTOM_MODELS_TIMEOUT` | 自定义模型端点探测超时（秒） | `5` |
| `HERMES_WEBUI_MAX_SESSIONS` | 最大会话数 | `1000` |
| `HERMES_WEBUI_CORS_ORIGIN` | CORS 允许来源 | 空 |
| `HERMES_WEBUI_LOG_LEVEL` | 日志级别 | `info` |
| `HERMES_WEBUI_DEBUG` | 调试模式（`true` 开启） | 空 |
| `HERMES_WEBUI_TLS_CERT` / `HERMES_WEBUI_TLS_KEY` | TLS 证书 / 密钥路径 | 空（关闭 TLS） |
| `HERMES_WEBUI_PASSKEY` | 启用 Passkey 登录（`true` 开启） | 空 |
| `HERMES_WEBUI_ONBOARDING_OPEN` | 开放引导流程（`1` 开启） | 空 |
| `HERMES_ALLOWED_ORIGINS` | CORS 允许的源（逗号分隔），生产环境必须配置 | 空 |
| `HERMES_KANBAN_TASK_ID` | 看板 worker 模式任务 ID（存在则进入 worker 模式，跳过 HTTP 服务） | 空 |

**重点环境变量说明：**

- `HERMES_WEBUI_PORT` / `HERMES_WEBUI_HOST`：控制服务监听地址。若需让局域网其他设备访问，可将主机设为 `0.0.0.0`（不安全，建议同时配置 `HERMES_ALLOWED_ORIGINS` 与密码）。
- `HERMES_WEBUI_CHAT_BACKEND`：用于指定聊天后端模式，可在自定义部署中切换后端实现。
- `HERMES_ALLOWED_ORIGINS`：生产环境必须配置，未配置时仅允许 `localhost` 与 `127.0.0.1` 各端口的跨站访问，防止任意源带 cookie 跨站访问。
- `HERMES_WEBUI_PLUGINS_DIR`：指定插件目录路径，未设置时使用默认的 `$状态目录/plugins`。
- `HERMES_KANBAN_TASK_ID`：当该变量存在时，程序进入看板 worker 模式，加载指定任务 → 调用 LLM → 标记 `done`/`blocked` → 退出，跳过 HTTP 服务器启动。

### 2.6 Profile 系统

Profile（配置文件）系统用于维护多套独立的智能体配置，实现不同角色或场景的隔离。

- 每个 Profile 拥有独立配置目录，位于 `$HERMES_HOME/profiles/<name>/`（`default` 直接使用 `$HERMES_HOME`）。
- 当前活跃 Profile 记录在 `$HERMES_HOME/active_profile` 文件中。
- 每个 Profile 目录下包含独立的 `config.yaml`、`.env`、`memories/`、`sessions/`、`skills/`、`logs/`、`plans/`、`workspace/`、`cron/` 等子目录。
- 可在前端界面创建、切换、删除 Profile，实现多套配置互不干扰。
- Profile 名称规则：仅允许小写字母、数字、连字符和下划线（正则 `^[a-z0-9][a-z0-9_-]{0,63}$`），且不能与内置的 `default` 重名。
- 创建 Profile 时可选择「复制当前配置档的配置」，将源 Profile 的 `config.yaml`、`.env`、`SOUL.md` 一次性克隆到新目录。
- 切换 Profile 通过设置 `hermes_profile` cookie（30天有效）实现 per-client 隔离，不同浏览器可同时使用不同 Profile。
- 删除 Profile 时会同时删除数据库记录与磁盘上的 Profile 目录，且不能删除当前活跃的 Profile。

**Profile 与工作区的区别**：Profile 定义智能体的工作方式（身份、记忆、技能、模型配置）；工作区定义它处理哪些文件（磁盘上的项目目录）。Profile 回答「谁在工作」，工作区回答「在哪里工作」。

### 2.7 访问Web界面

服务启动后，在浏览器中打开默认地址：

```
http://127.0.0.1:8787
```

若设置了密码（通过环境变量 `HERMES_WEBUI_PASSWORD` 或在「设置 → 系统」中配置），将先进入登录页面，输入密码后方可使用。首次访问若引导流程未关闭，会自动进入「首次使用」引导向导。

## 3. Web界面总览

Web界面采用三栏式布局：

- **左侧栏**：包含「导航栏（Rail）」与「侧边栏（Sidebar）」两部分。导航栏为窄竖条，放置12个主导航按钮；侧边栏根据当前选中的导航展示对应内容（如会话列表、看板列表、技能列表等）。侧边栏宽度可通过分隔条拖动调整，并支持折叠。
- **主内容区（Main）**：位于中央，根据当前导航显示对应的主视图（如聊天消息流、技能详情、记忆编辑器、设置面板等）。顶部可显示更新横幅、重连横幅、离线横幅、网关健康横幅等状态条。
- **工作区面板（右侧面板）**：默认隐藏，可通过聊天输入框旁的文件夹按钮或界面右缘的箭头按钮展开。包含「文件」「Artifacts」「待办」三个标签页，用于浏览工作区文件、查看产物与任务列表。面板宽度同样可拖动调整。

界面顶部为应用标题栏，包含品牌图标、（可选的）Profile 切换器、主题切换等入口。整体支持深色/浅色/跟随系统三种主题，以及多种皮肤。

界面所有文案均支持国际化，默认语言为简体中文（`zh`），可在「设置 → 偏好」中切换。

## 4. 侧边栏导航详解

导航栏（Rail）位于界面最左侧，包含12个导航按钮，点击后切换主视图与侧边栏内容。在窄屏设备上，导航栏会折叠为侧边栏顶部的图标行。各按钮可通过「设置 → 外观 → 侧边栏标签」隐藏或重新排序（聊天与设置始终可见）。

以下为12个导航按钮的详细说明，顺序与界面默认排列一致：

### 4.1 聊天（chat）

进入AI对话主界面。侧边栏显示会话列表，支持搜索、置顶、归档、重命名、删除、新建等操作。主内容区显示消息流与输入框。这是最常用的功能入口。

### 4.2 任务（tasks）

进入定时任务（Cron）管理界面。侧边栏显示所有定时任务列表，可新建、刷新、暂停、恢复、立即运行任务。详见 [第13章 Cron定时任务](#13-cron定时任务)。

### 4.3 看板（kanban）

进入看板任务管理界面。侧边栏显示看板任务列表，支持按负责人、租户筛选，包含已归档与仅我的过滤。可新建任务、批量操作、预览派发器、运行派发器。详见 [第8章 看板功能详解](#8-看板功能详解)。

### 4.4 技能（skills）

进入技能管理界面。侧边栏显示已加载的技能列表，支持搜索、新建、编辑、删除、启用/禁用。主内容区显示技能详情与 `SKILL.md` 编辑器。详见 [第9章 技能系统详解](#9-技能系统详解)。

### 4.5 记忆（memory）

进入记忆管理界面。侧边栏显示记忆栏目（个人记忆、用户资料、智能体灵魂、项目上下文、第三方笔记等），主内容区显示对应记忆文件的编辑器。详见 [第10章 记忆系统详解](#10-记忆系统详解)。

### 4.6 工作区（workspaces）

进入工作区管理界面。侧边栏显示已添加的工作区列表，可添加、切换、重命名、移除工作区。主内容区显示工作区详情。详见 [第7章 工作区面板详解](#7-工作区面板详解)。

### 4.7 配置文件（profiles）

进入Profile管理界面。侧边栏显示所有Profile卡片，展示其状态、模型、供应商、基础 URL、API 密钥、技能数、网关状态等信息，可创建、切换、删除Profile。详见 [第2.6节 Profile 系统](#26-profile-系统)。

### 4.8 待办（todos）

进入当前任务列表界面。显示当前会话中的活跃待办列表，便于跟踪本次对话产生的待办事项。

### 4.9 洞察（insights）

进入统计洞察界面。可按 7 天 / 30 天 / 90 天 / 365 天周期查看 token 用量、会话数等统计数据。需开启「同步到洞察」设置后才会包含 WebUI 浏览器会话的数据。

### 4.10 仪表盘（dashboard）

打开官方仪表盘链接。该按钮默认隐藏，需在「设置 → 系统 → 官方仪表盘」中配置为「自动检测」或「始终显示」后才会出现。点击后在新标签页打开仪表盘地址。公开反向代理 URL 仅作为浏览器链接存储，不会被服务器探测。

### 4.11 日志（logs）

进入日志查看界面。可选择日志文件（智能体、错误、网关）、尾部行数（100/200/500/1000）、严重级别（全部/错误/警告+），支持自动刷新（5秒）、自动换行、复制全部。详见 [第14章 终端面板](#14-终端面板) 中的日志说明。

### 4.12 设置（settings）

进入设置界面。侧边栏显示8个设置分区菜单与设置搜索框，主内容区显示对应分区的设置项。详见 [第6章 设置页面详解](#6-设置页面详解)。

## 5. 聊天界面详解

聊天界面是项目最核心的功能区域，由消息区、状态横幅、输入框（Composer）三部分组成。

### 5.1 消息区

- **空状态**：未选择会话时显示欢迎界面，包含三个快捷建议按钮（如「这个工作区里有哪些文件？」「我今天有什么安排？」「帮我规划一个小项目。」），点击即可快速发起对话。
- **消息流**：用户消息与助手消息交替显示，助手消息支持 Markdown 渲染（含代码块、表格、数学公式）、思考过程折叠、工具调用卡片、流式输出。
- **会话跳转按钮**：阅读长会话历史时，左侧会出现「开头」与「结尾」浮动按钮，便于快速跳转。需在「设置 → 外观」中开启「显示会话跳转按钮」。
- **向上加载更早消息**：开启「向上滚动时加载更早的消息」后，向上滚动会自动加载历史消息；关闭时需手动点击加载按钮。
- **自动跟随新内容**：开启后视图会随新 token 流入自动滚动到底部；关闭后由用户手动控制滚动位置。
- **会话大纲**：开启后显示仅限桌面的跳转列表，可快速跳转到当前会话中的用户问题。
- **消息操作**：助手消息支持复制、重新生成、编辑、用所选内容回复等操作；用户消息支持编辑后重新发送。

### 5.2 状态横幅

主内容区顶部可能显示以下状态横幅：

- **更新横幅**：检测到新版本时显示，提供「稍后」「立即更新」「强制更新」按钮，并可展开「新功能」摘要。
- **重连横幅**：上次离开时有回复正在生成时显示，提示重新加载消息。
- **离线横幅**：浏览器报告设备离线或无法访问服务时显示，恢复后自动刷新。
- **网关健康横幅**：网关心跳失败时显示，提供「重启服务」与「忽略」按钮。

### 5.3 输入框区域（Composer）

输入框区域是发送消息与控制会话的核心控件集合，从左到右、从上到下主要包含：

**输入区上方：**

- **排队卡片**：忙碌输入模式为「队列」时，发送的消息会进入排队，显示为可取消的卡片。
- **审批卡**：智能体请求执行危险命令时弹出，提供「允许一次」「允许本次会话」「始终允许」「拒绝」「跳过全部（YOLO）」按钮。详见 [第16章 模态框与对话框](#16-模态框与对话框)。
- **澄清卡**：智能体需要澄清时弹出，提供选项与自定义输入框。
- **终端面板**：嵌入式终端，可折叠/展开/停靠。详见 [第14章 终端面板](#14-终端面板)。

**输入框主体：**

- **附件托盘**：显示已添加的附件缩略图，支持拖拽文件到输入框添加附件。
- **麦克风状态**：语音输入时显示「聆听中…」。
- **语音模式条**：免手语音模式下显示状态指示。
- **消息输入框**：多行文本输入区域，占位符为「Message Hermes…」。

**输入框底部工具栏（左侧控件组）：**

- **附件按钮**（回形针图标）：打开文件选择对话框添加附件，支持图片、文本、PDF、JSON、Excel、Word、Markdown、代码、压缩包等多种类型。
- **保存的提示按钮**（书签图标）：打开已保存提示的弹出菜单，可保存/删除/插入常用提示。
- **麦克风按钮**：语音听写（需浏览器支持，默认隐藏）。
- **语音模式按钮**：免手语音模式（音频波形图标，需浏览器支持语音识别与TTS，默认隐藏）。
- **YOLO 模式胶囊**：开启后跳过所有审批，显示闪电图标与「YOLO」文字。
- **Profile 切换胶囊**：显示当前Profile名称，点击下拉切换Profile。
- **工作区控件组**：包含「显示工作区面板」按钮与「切换工作区」胶囊，胶囊显示当前工作区名称。
- **移动端配置按钮**：窄屏显示，环形显示上下文用量，点击展开工作区/模型/推理/上下文设置。
- **模型选择器**：显示当前模型名称，点击下拉切换模型。下拉按供应商分组，支持搜索，支持自定义模型 ID。
- **供应商配额标签**：显示当前供应商剩余配额（如 OpenRouter 信用余额），点击跳转到设置页供应商分区。
- **推理强度选择器**：选择推理努力级别（无 / 最小 / 低 / 中 / 高 / 超高），控制思考过程强度（默认隐藏，模型支持时显示）。
- **会话工具集选择器**：限制本会话可用的工具（留空=使用全局配置），显示「全局」或自定义工具列表。

**输入框底部工具栏（右侧控件组）：**

- **状态文本**：显示当前状态（如「正在响应…」）。
- **上下文指示器**：环形进度环显示上下文窗口用量百分比，悬停显示详细用量、token 数、阈值、成本，并提供「立即压缩」按钮。
- **后台任务徽章**：有后台任务运行时显示数量。
- **发送按钮**：发送消息（箭头图标），无内容或正在生成时禁用。

### 5.4 会话管理

在侧边栏会话列表中可进行以下操作：

- **新建会话**：点击侧边栏顶部的「+」按钮，或使用快捷键 `Cmd+K`（macOS）/ `Ctrl+K`（Windows）。
- **切换会话**：点击列表中的会话项。
- **搜索会话**：在顶部搜索框输入关键词筛选会话。
- **重命名会话**：双击会话项，或在右键菜单选择「重命名对话」。
- **重新生成标题**：让智能体根据对话内容生成新的标题。
- **复制对话链接**：复制当前对话的直接链接。
- **置顶会话**：将会话置顶显示（数量受「置顶对话数量限制」设置控制，默认3）。
- **归档会话**：将会话移入归档。
- **清空对话**：清空当前对话的所有消息（不可撤销）。
- **删除会话**：从列表中删除。
- **导出**：将会话导出为 Markdown 转录文本或 JSON 文件。
- **导入**：从 JSON 文件导入会话。
- **重试 / 撤销**：对最近一轮消息进行重试或撤销。
- **复制到工作树**：为该工作区创建一个隔离的 git worktree 会话。

会话列表支持「显示外部会话」（来自 CLI、飞书、钉钉、微信、WhatsApp、Telegram、Discord、Slack 等渠道）与「显示定时任务会话」选项，可在「设置 → 偏好」中开启。

## 6. 设置页面详解

点击侧边栏导航的「设置」按钮进入设置页面。侧边栏显示8个分区菜单与设置搜索框（可在搜索框中输入关键词快速定位设置项），主内容区显示对应分区的设置项。多数设置项修改后会自动保存并显示「已保存」状态；部分分区底部提供「保存设置」按钮。

### 6.1 会话（Conversation）

会话分区显示当前活动会话的元信息，并提供以下会话级操作按钮：

| 按钮 | 说明 |
|------|------|
| 转录文本 | 将当前会话下载为 Markdown 文件 |
| JSON | 将会话完整导出为 JSON 文件 |
| 导入 | 从 JSON 文件导入会话 |
| 清空 | 清空当前对话中的所有消息（不可撤销） |

此外，与会话行为相关的若干设置项分布在其他分区或后端默认值中：

- **自动加载记忆**（`auto_load_memory`，默认 `true`）：是否在会话开始时自动加载记忆文件。该设置由记忆系统控制，详见 [第10章 记忆系统详解](#10-记忆系统详解)。
- **显示思考过程**（`show_thinking`，默认 `true`）：是否显示推理模型的思考过程。
- **最大迭代次数**（`max_iterations`，默认 `0`，范围 0-200）：单轮对话中智能体的最大迭代次数，0 表示不设限。
- **AI 请求超时**（`ai_request_timeout`，默认 `600`，范围 30-1800 秒）：用于服务商请求、流式首个响应及连接测试的超时时间。部分服务商在大上下文下可能需要数分钟。该设置位于「偏好」分区。
- **简化工具调用**（`simplified_tool_calling`，默认 `true`）：是否使用简化版的工具调用流程。
- **忙碌输入模式**（`busy_input_mode`，默认 `queue`）：控制智能体运行时发送消息的行为，位于「偏好」分区。可选「队列后续消息」「中断当前轮次」「引导（中间修正）」。

### 6.2 外观（Appearance）

外观分区用于控制界面的视觉风格与布局，所有修改自动保存。

**主题（Theme）**：三选一卡片式选择。

**皮肤（Skin）**：在主题基础上提供多种配色风格，包括 `default`、`ares`、`mono`、`graphite`、`slate`、`poseidon`、`sisyphus`、`charizard`、`sienna`、`catppuccin`、`nous`、`geist-contrast`、`zeus`、`verdigris` 等。也可通过斜杠命令 `/theme` 切换。

**字体大小（Font size）**：四选一卡片式选择。

| 选项 | 说明 |
|------|------|
| Small（小） | 较小字体 |
| Default（默认） | 默认字体（默认选中） |
| Large（大） | 较大字体 |
| Extra Large（超大） | 超大字体 |

**其余外观设置项（均为复选框）：**

| 设置项 | 默认 | 说明 |
|--------|------|------|
| 默认保持工作区面板打开 | 关 | 新会话时自动打开工作区/文件浏览面板 |

| 自动展开工作日志详情 | 开 | 新工作日志详情默认展开 |

**活动显示方式（Activity display）**：二选一切换按钮。

| 选项 | 说明 |

**侧边栏标签（Sidebar tabs）**：通过可拖拽的标签卡片控制哪些导航标签显示及顺序。聊天与设置始终可见，不可隐藏。

### 6.4 供应商（Providers）

供应商分区用于管理 AI 模型供应商的 API 密钥与配置，更改立即生效。每个供应商卡片显示其配置状态（已配置 API 密钥 / OAuth / 未配置），并提供以下操作：

- **编辑设置**：打开 API 密钥输入框与模型刷新。
- **保存**：保存 API 密钥。
- **移除**：移除 API 密钥。
- **刷新模型**：重新拉取该供应商的可用模型列表。

项目通过 `providers.json` 内置支持以下供应商类型：

| 供应商 | 说明 |
|--------|------|
| `deepseek` | DeepSeek 官方接口（含推理模型） |
| `硅基流动` | 硅基流动聚合平台（数十个模型可选） |
| `MiMo` | MiMo 模型接口 |
| `custom` | 任意 OpenAI 兼容接口（自定义 `base_url` 与 `api_key`） |

供应商的 API Key、Base URL 等也可直接编辑 `providers.json`。健康检查接口（`/health`）会探测各供应商的可达性与延迟。

创建 Profile 时可填写 `base_url`、`api_key`、`model_provider`、`default_model`，系统会自动写入对应 Profile 的 `config.yaml` 与 `.env`。API Key 会根据供应商类型映射到对应的环境变量名（如 `DEEPSEEK_API_KEY`、`OPENAI_API_KEY`、`ANTHROPIC_API_KEY`、`GEMINI_API_KEY` 等），未匹配时使用通用的 `HERMES_API_KEY`。

供应商分区还显示「当前提供商配额」信息，包括 5 小时限制、每周限制、剩余/已用/限额、重置时间等（部分供应商支持）。

### 6.5 插件（Plugins）

插件分区为只读面板，显示已安装的 插件及其注册的生命周期钩子。每个插件卡片显示：

- 插件名称（未命名时显示「未命名插件」）
- 描述（未提供时显示「未提供描述信息」）
- 启用状态（已启用 / 已禁用 / 已启用（provider））
- 已注册的挂钩列表（未注册时显示「未注册生命周期挂钩」）

若当前没有可见的插件，显示提示：「当前没有可见的 插件。请从 CLI/配置中安装或启用插件以在此处查看。」。插件的安装与启用需通过 CLI 或配置完成，详见 [第11章 插件系统详解](#11-插件系统详解)。

### 6.6 扩展（Extensions）

扩展分区显示已配置的 WebUI 扩展的诊断信息与启用/禁用控制。

- **信任模型说明**：扩展运行在 WebUI 浏览器源（origin）中，可访问与本会话相同的已认证 API。仅加载受信任的本地扩展目录；禁用扩展对已注入的资源在重新加载后生效。
- **诊断信息**：显示各扩展的加载状态、错误等。
- **复制诊断**：将诊断信息复制到剪贴板（无扩展时按钮禁用）。

扩展通常用于集成 MCP、技能等高级能力。

### 6.7 系统（System）

系统分区显示实例版本与访问控制，是设置中信息量较大的分区。

**版本与更新检查**：

- 显示 WebUI 版本徽章与 Agent 版本徽章（如「WebUI: 1.0.3」「Agent: not detected」）。
- 「立即检查」按钮手动检查更新，显示「检查中…」「已是最新」「有 N 个更新可用」「更新检查已禁用」「更新检查失败」等状态。

**访问密码**：

- 输入新密码以设置或更改密码，留空保持当前设置。
- 若已启用密码，显示「当前密码」输入框用于验证。
- 若环境变量 `HERMES_WEBUI_PASSWORD` 已设置，则密码输入框被锁定并提示需移除环境变量并重启服务器才能在此管理密码。
- 未启用认证时显示「未认证」警告，提示此实例可被无认证访问，并提供「我已了解此风险」复选框以降低导航栏警告的紧迫度。
- 「停用认证」按钮移除密码保护。
- 「退出登录」按钮登出当前会话。

**停止服务器**：

- 「停止服务器」按钮优雅地停止本地 WebUI 服务器。停止后需重新启动服务器（火山服务器软件开发平台运行或原生应用）才能再次访问 WebUI。

**Passkey（通行密钥）**：

- 注册当前浏览器或设备用于无密码登录。
- 至少存在一个 Passkey 后可移除密码，仅用 Passkey 登录。
- 显示已注册的 Passkey 列表（无则显示「No passkeys registered.」）。
- 需环境变量 `HERMES_WEBUI_PASSKEY=true` 启用。

**官方仪表盘**：

- 下拉选择「自动检测」「始终显示」「从不显示」，控制导航栏是否显示仪表盘链接。
- 可填写仪表盘 URL（如 `http://127.0.0.1:9119`）。
- 「保存仪表盘链接设置」按钮保存配置。

**网关状态**：

- 显示网关（飞书、钉钉、微信、WhatsApp、Telegram、Discord、Slack 等）的运行状态、最后活跃时间、会话数。
- 状态包括：运行中、未运行、未配置、元数据已过时、端点无法访问。

**MCP 服务器**：

- 显示 `config.yaml` 中配置的 MCP 服务器列表。
- 每个服务器显示状态（运行中、已配置、已禁用、配置无效、已连接、连接失败、检查中、需授权、未知）与工具数。
- 提供「编辑」「重新连接」「移除」操作。
- 提示：服务器更改保存到 `config.yaml`，重启后生效。

**MCP 工具**：

- 搜索活跃 MCP 服务器中的已知工具。
- 显示工具总数、已连接数、需关注数等统计。
- 支持按名称、服务器或描述搜索，分页浏览。
- 提示：工具清单仅使用已知的活跃 MCP 运行时数据；WebUI 不会启动或探测服务器。

> 关于端口、主机、后端模式等系统级配置：这些不在Web界面中直接修改，而是通过环境变量（如 `HERMES_WEBUI_PORT`、`HERMES_WEBUI_HOST`、`HERMES_WEBUI_CHAT_BACKEND`）或 `config.yaml` 配置。详见 [第2.5节 环境变量](#25-环境变量)。

### 6.8 帮助（Help）

帮助分区提供资源与支持入口，包含两张帮助卡片：

| 卡片 | 说明 |
|------|------|
| 文档 | 打开官方文档（指南、配置参考、完整 README） |
| 反馈问题 | 在 GitHub 搜索已有问题或提交新问题 |

帮助分区还内置「使用指南」说明，简要介绍各主要功能按钮的用法（对话、模型选择、推理/工具、设置、供应商、IM 适配器、插件）。

## 7. 工作区面板详解

工作区面板位于界面右侧，默认隐藏，可通过以下方式打开：

- 聊天输入框底部工具栏的「显示工作区面板」按钮（文件夹图标）。
- 界面右缘的箭头按钮。
- 在「设置 → 外观」中开启「默认保持工作区面板打开」，使新会话自动打开。

面板顶部为标题栏，显示当前工作区名称、Git 徽章（若在工作区中）、隐藏文件可见指示器，以及一组操作按钮：

| 按钮 | 说明 |
|------|------|
| 上级目录 | 返回上级目录（仅在子目录中显示） |
| 新建文件 | 在当前目录新建文件 |
| 新建文件夹 | 在当前目录新建文件夹 |
| 刷新 | 刷新工作区面板 |
| 上传文件 | 上传文件到当前目录 |
| 工作区选项 | 打开选项菜单（含显示隐藏文件等） |
| 关闭预览 | 关闭文件预览区 |

面板下方为三个标签页：

### 7.1 文件树操作（Files 标签页）

- **浏览**：以树形结构显示工作区文件与文件夹，点击文件夹展开/折叠，点击文件在下方预览区打开。
- **面包屑导航**：显示当前路径，可点击跳转。
- **新建**：通过「新建文件」「新建文件夹」按钮，输入名称即可创建。
- **重命名**：双击文件/文件夹名称重命名。
- **删除**：右键菜单或操作按钮删除（删除文件夹需二次确认）。
- **上传**：通过「上传文件」按钮选择文件，或直接拖拽文件到面板上传（受 `max_upload_mb` 限制，默认 50MB）。上传过程显示实时字节进度条，支持多文件顺序上传。
- **下载**：在预览区点击「下载」按钮下载文件到本地。
- **移动**：右键菜单移动文件/文件夹到其他位置。
- **在文件管理器中显示**：右键菜单在系统文件管理器中定位。
- **复制文件路径**：右键菜单复制绝对路径。
- **在 VS Code 中打开**：右键菜单用 VS Code 打开。
- **下载文件夹**：右键菜单下载整个文件夹。
- **显示隐藏文件**：通过「工作区选项」菜单切换是否显示 `.DS_Store`、`.git`、`node_modules` 等隐藏/系统文件。

### 7.2 预览区

点击文件后，预览区根据文件类型展示内容：

- **代码文件**：以语法高亮显示代码。
- **图片**：显示图片预览。
- **PDF**：内嵌 iframe 预览 PDF。
- **Markdown**：渲染为富文本（大文件可强制渲染）。
- **HTML**：在沙盒 iframe 中渲染（允许脚本、弹窗）。
- **音视频**：内嵌播放器。
- **编辑**：点击「编辑」按钮进入编辑模式，修改后保存。

预览区顶部显示文件路径、徽章，以及「在浏览器中打开」「下载」「编辑」等操作按钮。

### 7.3 Artifacts

Artifacts 标签页显示当前会话产生的产物（如智能体生成的代码、文档等），徽章显示数量。点击可在预览区查看或编辑。

### 7.4 待办（Todos）

待办标签页显示当前会话中的活跃待办列表（需在「设置 → 外观」中开启「在工作区面板显示待办标签」）。与侧边栏的「待办」导航功能一致，便于在工作时不切换面板查看任务。

### 7.5 工作区管理

工作区列表（侧边栏「工作区」导航）用于管理多个工作区：

- **添加工作区**：点击「+」按钮，输入绝对路径（保存前会校验路径是否为已存在目录）。
- **切换工作区**：点击列表项切换当前会话使用的工作区。智能体运行中时无法切换。
- **重命名工作区**：编辑工作区显示名称。
- **移除工作区**：从列表移除（不删除磁盘文件）。
- **拖动重排序**：拖动列表项调整顺序。
- **新建工作树会话**：为该工作区创建一个隔离的 git worktree 会话。

切换工作区若有未保存的文件编辑，会提示放弃。

## 8. 看板功能详解

看板（Kanban）是内置的任务工作流系统，用于管理智能体的工作队列。

### 8.1 看板视图

进入看板导航后，侧边栏显示看板筛选与任务列表，主内容区显示看板视图。侧边栏顶部提供：

- **搜索框**：按关键词搜索任务。
- **负责人筛选**：按任务负责人筛选（包含 Profile 与其他 CLI 范围/已移除的 Profile）。
- **租户筛选**：按租户（项目或团队 slug）筛选。
- **包含已归档**：复选框，是否显示已归档任务。
- **仅我的**：复选框，仅显示分配给当前 Profile 的任务。
- **统计**：显示任务数量统计。
- **批量操作栏**：包含状态批量更新下拉、「批量操作」「预览派发器」「运行派发器」按钮。
- **新建任务行**：快速输入任务标题并回车创建。

主内容区可创建多个看板，每个看板有独立的名称、slug、描述、图标、颜色。看板支持「重命名当前看板」「归档当前看板」操作（归档后任务保留在磁盘 `kanban/boards/_archived/`，可恢复）。

### 8.2 任务状态

看板任务有以下状态，按工作流流转：

| 状态 | 中文 | 说明 |
|------|------|------|
| triage | 分流 | 新建任务的初始状态，待分类 |
| todo | 待办 | 已分类待执行 |
| ready | 就绪 | 准备好可被派发器认领 |
| running | 运行中 | worker 正在执行 |
| blocked | 已阻塞 | 因某种原因阻塞 |
| done | 已完成 | 执行完成 |
| archived | 已归档 | 已归档（不显示在常规视图） |

任务编辑对话框支持 `分流 / 待办 / 就绪` 状态编辑，其他状态由系统自动流转。任务卡支持「完成」「归档」操作。

### 8.3 任务创建、分配、执行

**新建任务**：点击「新建任务」按钮或在新任务输入行输入标题回车。任务编辑对话框包含以下字段：

| 字段 | 说明 |
|------|------|
| 标题 | 必填 |
| 描述 | 可选，说明需要做什么、验收标准、链接 |
| 状态 | 分流 / 待办 / 就绪 |
| 负责人 | 可选，留空表示任何 worker |
| 租户 | 可选，项目或团队的 slug |
| 优先级 | 数字越大越先执行，默认 0 |
| 评论 | 可添加评论 |
| 链接 / 父任务 / 子任务 | 关联其他任务 |

**分配**：通过「负责人」字段分配给特定 Profile。若选择「未分配」+「就绪」，派发器将跳过此任务（需重新提交确认）。

**执行**：

- **运行派发器**：认领此看板上的就绪任务并生成 worker 子进程（每个任务一个，每次点击最多 8 个）。点击前会弹出确认。
- **预览派发器**：试运行，显示会被认领的任务而不实际生成 worker。
- 派发结果会显示：已启动、已提升、已认领、已跳过（未分配）、已跳过（未知 Profile）、自动阻塞、已超时、崩溃等。

**Worker 模式**：当环境变量 `HERMES_KANBAN_TASK_ID` 设置时，程序进入 worker 模式，加载指定任务 → 调用 LLM → 标记 `done`/`blocked` → 退出。此模式下不启动 HTTP 服务器。

### 8.4 批量操作

在批量操作栏中：

- 选择目标状态（就绪 / 已阻塞 / 已完成 / 已归档）。
- 点击「批量操作」将选中任务批量更新为该状态。
- 「预览派发器」与「运行派发器」可对当前筛选结果进行批量派发。

任务详情视图可查看评论、事件、链接、父/子任务、运行记录、Worker 日志等，并支持「阻塞」「解除阻塞」「返回看板」「按配置文件分泳道」等操作。

## 9. 技能系统详解

技能（Skills）是基于 `SKILL.md` 元数据组织的可复用能力单元，可被智能体在对话中调用。

### 9.1 技能目录结构

技能文件存放于每个 Profile 的 `skills/` 子目录下（默认 Profile 位于 `$HERMES_HOME/skills/`，其他 Profile 位于 `$HERMES_HOME/profiles/<name>/skills/`）。每个技能通常是一个目录，包含：

- `SKILL.md`：技能主文件，包含 YAML 前置元数据与 Markdown 正文。
- 其他辅助文件（脚本、资源等）。

### 9.2 SKILL.md 元数据

`SKILL.md` 文件由两部分组成：

**YAML 前置元数据**（front matter）：定义技能的元信息，常见字段包括：

| 字段 | 说明 |
|------|------|
| `name` | 技能名称（必填，英文标识符） |
| `category` | 分类（可选，如 `devops`） |
| `description` | 描述 |
| 其他 | 触发条件、参数、权限等 |

**Markdown 正文**：技能的具体内容、指令、示例等，智能体调用技能时会读取该部分作为上下文。

在Web界面「技能」导航的技能详情页可直接编辑 `SKILL.md` 的元数据与内容，保存后生效。注意：不支持重命名技能，如需重命名请新建并删除旧的。

### 9.3 技能热重载

项目内置技能热重载监视器，启动时自动启动（技能变更检测器）。当 `skills/` 目录下的文件发生变化（新增、修改、删除）时，系统会自动重新加载技能，无需重启服务。在Web界面编辑技能保存后即刻生效。

### 9.4 斜杠命令

在聊天输入框中输入 `/` 可触发斜杠命令下拉，显示可用命令。技能相关的斜杠命令包括：

| 命令 | 说明 |
|------|------|
| `/skills` | 列出可用技能 |
| `/use <技能名>` | 强制在下次消息中使用某个技能 |
| `/help` | 显示可用的斜杠命令 |
| `/clear` | 清空对话消息 |
| `/compress [topic]` | 手动压缩上下文 |
| `/model <名称>` | 切换模型 |
| `/workspace <名称>` | 按名称切换工作区 |
| `/terminal` | 打开工作区终端 |
| `/new` | 开始新聊天会话 |
| `/usage` | 切换 token 使用量显示 |
| `/theme` | 切换外观（主题与皮肤） |
| `/personality` | 切换智能体人格 |
| `/goal <文本>` | 设置或查询持久目标 |
| `/queue <消息>` | 将消息排队到下一轮 |
| `/interrupt <消息>` | 取消当前轮次并发送新消息 |
| `/steer <消息>` | 在不中断的情况下向当前轮次注入修正 |

技能启用/禁用可在技能列表中通过开关切换，禁用的技能不会被智能体调用。

## 10. 记忆系统详解

记忆系统通过 Markdown 文件维护智能体的人格、用户画像与跨会话记忆，存放在每个 Profile 的 `memories/` 子目录下。

### 10.1 记忆文件

| 文件 | 说明 |
|------|------|
| `SOUL.md` | 智能体灵魂，定义智能体的人格、行为准则、语气风格等。尚未定义时显示「尚未定义 Soul。」 |
| `USER.md` | 用户资料，记录用户的偏好、背景、习惯等信息。 |
| `MEMORY.md` | 记忆（备注），跨会话持久的事实与笔记。 |
| 项目上下文 | 当前工作区的项目上下文文件，未找到时显示「未找到此工作区的项目上下文文件。」 |

在Web界面「记忆」导航的侧边栏中选择对应栏目，即可在主内容区编辑相应文件。编辑后保存即生效，跨会话持久。

### 10.2 自动加载记忆

`auto_load_memory` 设置（默认 `true`）控制是否在会话开始时自动加载记忆文件。开启后，每次新会话智能体会自动读取 `SOUL.md`、`USER.md`、`MEMORY.md` 等记忆文件作为上下文，使智能体具备持续的记忆能力。该设置为后端默认值，可通过数据库 settings 表或环境变量调整。

### 10.3 第三方笔记

记忆面板还支持第三方笔记源（Joplin、Obsidian、Notion、llm-wiki 等）。配置后会在侧边栏显示对应的笔记源栏目，可搜索与预览笔记。自动会话召回保持不变；面板只显示已配置的来源和可用的读取/搜索工具。若尚未配置任何笔记源，显示「还没有可见的笔记或知识 MCP 来源」。

## 11. 插件系统详解

插件系统用于扩展的能力，通过生命周期钩子介入智能体的运行流程。

### 11.1 插件目录

插件默认存放于 `$状态目录/plugins/`（即 `$HERMES_HOME/webui/plugins/`），可通过环境变量 `HERMES_WEBUI_PLUGINS_DIR` 覆盖。服务启动时会自动加载该目录下的插件。

### 11.2 manifest.json

每个插件通常包含一个 `manifest.json`（或等价的配置），声明插件名称、描述、入口、注册的生命周期钩子等。Web界面的「设置 → 插件」分区会读取这些信息以只读方式展示。

### 11.3 插件启用/禁用

插件的启用/禁用通过 CLI 或配置文件完成（Web界面插件面板为只读）。已启用插件显示「已启用」，未启用显示「已禁用」，provider 类插件显示「已启用（provider）」。

### 11.4 插件卡片显示

在「设置 → 插件」分区，每个插件卡片显示：

- 插件名称（未命名时显示「未命名插件」）
- 描述（未提供时显示「未提供描述信息」）
- 启用状态徽章
- 已注册的生命周期挂钩列表（未注册时显示「未注册生命周期挂钩」）

若没有可见插件，显示提示：「当前没有可见的 插件。请从 CLI/配置中安装或启用插件以在此处查看。」

插件加载失败时会在面板显示「加载插件失败：」错误信息。

## 12. MCP（模型上下文协议）

MCP（Model Context Protocol）用于扩展智能体的工具能力，通过配置 MCP 服务器接入外部工具、资源、提示。

### 12.1 MCP服务器配置

MCP 服务器在 `config.yaml` 中配置。在Web界面「设置 → 系统 → MCP 服务器」分区可查看已配置的服务器列表，并进行以下操作：

- **添加服务器**：点击「+ 添加服务器」打开添加对话框。
- **编辑**：修改服务器配置。
- **重新连接**：重新建立连接。
- **移除**：删除服务器（需二次确认）。

MCP 服务器配置字段：

| 字段 | 说明 |
|------|------|
| 服务器名称 | 必填，需符合 `^[a-zA-Z0-9_-]+$` |
| 传输类型 | Stdio / HTTP / SSE |
| 命令 | stdio 传输的可执行命令 |
| 参数 | 命令参数（空格或逗号分隔） |
| URL | HTTP/SSE 传输的地址（必填） |
| 环境变量 | 键值对 |
| 请求头 | HTTP/SSE 的请求头 |
| 超时（秒） | 连接/请求超时 |
| 已启用 | 是否启用 |
| 作用域 | 项目 / 本地 / 用户 / 其他 |
| OAuth | OAuth 客户端 ID、回调端口等（高级） |

服务器状态包括：运行中、已配置、已禁用、配置无效、已连接、连接失败、检查中、需授权、未知。服务器更改保存到 `config.yaml`，需重启后生效。

### 12.2 工具调用

「设置 → 系统 → MCP 工具」分区可搜索活跃 MCP 服务器中的已知工具：

- 搜索框支持按名称、服务器或描述搜索。
- 显示工具总数、已连接数、需关注数等统计。
- 工具详情可查看信息、工具、资源、提示等标签页。
- 支持分页浏览（可设置每页数量）。

智能体在对话中可调用已连接 MCP 服务器提供的工具。工具清单仅使用已知的活跃 MCP 运行时数据，WebUI 不会主动启动或探测服务器。

## 13. Cron定时任务

定时任务（Cron）用于按调度自动执行智能体任务或脚本。

### 13.1 任务创建

在「任务」导航中点击「+」按钮打开创建对话框，填写以下字段：

| 字段 | 说明 |
|------|------|
| 任务名称 | 任务标识 |
| 调度 | 必填，cron 表达式（如 `0 9 * * *`）或自然语言（如 `every 1h`） |
| 模式 | 智能体 / 脚本 |
| 提示词 | 智能体模式下必填，发送给智能体的提示 |
| 脚本路径 | 脚本模式下，解析到 `~/.hermes/scripts/`（除非绝对路径） |
| 工作目录 | 任务执行的工作目录 |
| 覆盖模型 | 可选，覆盖此任务的默认模型 |

**智能体模式**：按调度调用智能体执行提示词，可使用技能。**脚本模式**：按调度执行脚本，stdout 投递到目标，无 Agent、Prompt 或 Skills。

### 13.2 执行方式

任务状态包括：关闭、已暂停、错误、活跃、运行中…、需要关注。可执行以下操作：

| 操作 | 说明 |
|------|------|
| 立即运行 | 立即触发一次执行 |
| 暂停 | 暂停调度 |
| 恢复 | 恢复调度并重新计算下次执行时间 |
| 编辑 | 修改任务配置 |
| 删除 | 删除任务（不可撤销） |
| 查看完整输出 | 查看执行输出 |
| 所有执行 | 查看历史执行记录 |
| 复制诊断信息 | 复制任务诊断信息 |

任务执行完成后会显示完成状态（已完成 / 失败）。若任务没有下一次执行时间，显示「需要关注」，可能是调度器尚未计算或 Gateway 缺少 `croniter` 包。可点击「恢复并重新计算」或「立即运行一次」处理。

任务列表顶部可能显示 Gateway 通知（如缺少 `croniter` 包时提示用 cron 支持重启）。开启「显示定时任务会话」设置后，定时任务输出会显示为侧边栏会话。

## 14. 终端面板

终端面板是嵌入式终端，可在浏览器中执行命令，仅支持本地终端后端。已针对各平台做回显与编码适配：

- **Windows**：使用 `cmd.exe /Q /K` 管道模式，前端手动回显输入（管道模式不逐字符回显），GBK 输出自动转为 UTF-8 显示。
- **Linux / macOS**：使用 `bash -i` / `zsh -i` 交互模式，由 shell 自身逐字符回显，前端不再重复回显，避免「输入一个字显示两个字」。
- **键盘与鼠标粘贴**：支持 `Ctrl+V` 键盘粘贴与鼠标右键粘贴，均已处理重复输入问题。

### 14.1 打开终端

- 在聊天输入框底部，终端面板默认隐藏。
- 通过斜杠命令 `/terminal` 打开工作区终端。
- 命令有输出时若开启「终端有输出时自动展开」，折叠的终端面板会自动展开。

### 14.2 终端操作

终端面板顶部标题栏显示「终端 · 工作区名称」，并提供以下操作按钮：

| 按钮 | 说明 |
|------|------|
| 清屏 | 清空终端输出 |
| 复制输出 | 复制终端输出到剪贴板 |
| 重启 | 重启终端会话 |
| 折叠 | 折叠终端面板（停靠到底部） |
| 关闭 | 关闭终端面板 |

终端底部为命令输入框，占位符「输入命令...」。终端面板可通过分隔条调整高度。折叠后显示为停靠条，含「展开」「关闭」按钮。

终端启动失败、输入失败、复制失败等情况会显示相应错误提示。远程后端不支持嵌入式终端，仅支持本地终端后端。

### 14.3 日志查看

「日志」导航提供日志查看功能（与终端不同，用于查看文件日志）：

- **日志文件**：可选 `agent`（智能体）、`errors`（错误）、`gateway`（网关）。
- **尾部行数**：100 / 200 / 500 / 1000。
- **严重级别**：全部 / 错误 / 警告+。
- **自动刷新**：每 5 秒自动刷新（默认开启）。
- **自动换行**：长行自动换行。
- **复制全部**：复制全部日志内容。

日志为空时显示「暂无日志内容」，加载中显示「正在加载日志…」，大型日志文件会截断尾部以控制内存占用。

## 15. Git集成

工作区面板顶部显示 Git 徽章（`gitBadge`），当当前工作区为 Git 仓库时显示分支等信息。Git 集成主要用于：

- **工作树会话**：在会话列表右键「在 worktree 中新建会话」，为该工作区创建一个隔离的 git worktree，便于在不影响主分支的情况下实验。
- **工作树状态与移除**：通过 `/api/session/worktree/status` 与 `/api/session/worktree/remove` 接口管理工作树。
- **diff 查看**：智能体执行 git 相关操作后，可在消息中查看 patch 文件的 diff（支持树形与原始视图）。

Git 操作主要通过智能体在终端或会话中执行 git 命令完成，Web界面提供状态展示与工作树管理辅助。

## 17. 卸载

### 17.1 停止服务

在「设置 → 系统」中点击「停止服务器」按钮优雅停止本地 WebUI 服务器，或直接关闭火山服务器软件开发平台运行窗口/可执行文件进程。

### 17.2 数据清理

完整卸载需清理以下数据：

| 项目 | 路径 | 说明 |
|------|------|------|
| Home 目录（Windows） | `%LOCALAPPDATA%\hermes` | 包含 `config.yaml`、`active_profile`、`profiles/`、`webui/`（含 `sessions/`、`settings.json`、`httpServer.db` 等） |
| Home 目录（Linux/macOS/Android） | `~/.hermes` | 同上，各平台默认 Home 路径 |
| 旧版遗留目录（Windows） | `%USERPROFILE%\.hermes` | v0.51.134 之前的 Windows 默认位置，仅当新位置无状态而旧位置有状态时回退使用 |
| 项目目录 | 项目源码所在目录 | 项目源码与编译产物，可整体删除 |
| 编译产物 | 可执行文件同目录下的 `static/` 等 | 编译时复制的静态文件 |

**清理步骤：**

1. 停止服务。
2. 删除 Home 目录（Windows 为 `%LOCALAPPDATA%\hermes`，其他平台为 `~/.hermes`），含所有 Profile、会话、技能、记忆、数据库。
3. Windows 上若存在 `%USERPROFILE%\.hermes` 旧版目录且不再需要，一并删除。
4. 删除项目源码目录与编译产物。
5. 清理环境变量（如已设置 `HERMES_HOME`、`HERMES_WEBUI_PASSWORD` 等）。

> 删除 Home 目录将永久丢失所有会话、记忆、技能、配置与 Profile 数据，操作前请备份重要内容。

## 18. 常见问题（FAQ）

**Q1：启动后浏览器无法访问 `http://127.0.0.1:8787`？**

- 确认控制台已输出 `listening on http://127.0.0.1:8787`。
- 检查端口是否被占用，可通过 `HERMES_WEBUI_PORT` 换一个端口。
- 检查防火墙是否拦截。
- 若主机设为 `0.0.0.0`，仍应通过 `127.0.0.1` 或 `localhost` 本机访问。

**Q2：忘记登录密码怎么办？**

- 若密码通过环境变量 `HERMES_WEBUI_PASSWORD` 设置，移除该环境变量并重启服务即可清除密码。
- 若通过「设置 → 系统」设置，需直接编辑数据库 `httpServer.db` 的 `settings` 表，删除 `key='password_hash'` 的记录，或重新设置。

**Q3：模型列表为空或供应商不可达？**

- 在「设置 → 供应商」检查 API 密钥是否已配置。
- 点击「刷新模型」重新拉取。
- 检查 `providers.json` 配置是否正确。
- 访问 `/health` 接口查看各供应商可达性与延迟。
- 自定义端点需确保 `base_url` 以 `http://` 或 `https://` 开头，通常以 `/v1` 结尾。

**Q4：如何让局域网其他设备访问？**

- 设置 `HERMES_WEBUI_HOST=0.0.0.0` 重启服务（不安全）。
- 强烈建议同时配置 `HERMES_ALLOWED_ORIGINS` 为允许的源（逗号分隔）并设置访问密码。
- 通过 `http://<本机IP>:8787` 访问。

**Q5：配置修改后不生效？**

- 环境变量优先级高于数据库与 YAML，需检查是否被环境变量覆盖。
- `config.yaml` 修改后会自动重载（基于文件修改时间）。
- 数据库 settings 表通过Web界面「设置」保存的项优先级高于 YAML。
- 内存覆盖优先级最高，仅运行时生效，重启后失效。
- MCP 服务器配置需重启后生效。

**Q6：Profile 切换后模型/工作区未变化？**

- 切换 Profile 会读取该 Profile 的 `config.yaml` 中的 `model.default` 与 `model.provider`。
- 工作区读取该 Profile 的 `webui_state/last_workspace.txt`。
- 切换 Profile 后会新建对话以应用新配置。

**Q7：技能修改后未生效？**

- 技能支持热重载，保存后应即刻生效。
- 确认技能未被禁用（技能列表中的开关）。
- 确认 `SKILL.md` 的 YAML 前置元数据格式正确。

**Q8：看板派发器不认领任务？**

- 任务状态必须为「就绪」才会被认领。
- 任务必须已分配负责人（未分配的任务会被跳过）。
- 负责人对应的 Profile 必须存在（未知 Profile 会被跳过）。
- 每次点击最多生成 8 个 worker。

**Q9：终端无法使用？**

- 嵌入式终端仅支持本地终端后端，远程后端不支持。
- 确认工作区已选择（终端需绑定工作区）。
- 查看终端错误提示。

**Q10：如何完全重置为初始状态？**

- 停止服务。
- 删除 Home 目录（Windows 为 `%LOCALAPPDATA%\hermes`，Linux/macOS/Android 为 `~/.hermes`，或自定义的 `HERMES_HOME` 目录）。
- 重启服务，将进入首次使用引导流程。

**Q11：上传文件失败提示文件过大？**

- 上传大小受 `HERMES_WEBUI_MAX_UPLOAD_MB` 限制（默认 50MB）。
- 可通过环境变量调大该值后重启服务。
- 请求体总大小上限为 100MB。

**Q12：日志面板显示「暂无日志内容」？**

- 选择正确的日志文件（智能体/错误/网关）。
- 确认对应日志文件已写入内容（显示「尚未写入」表示文件为空）。
- 点击「刷新」按钮手动刷新。

**Q13：如何关闭更新检查横幅？**

- 在「设置 → 偏好」中关闭「检查更新」。
- 关闭后不再显示更新横幅，也不再执行后台 git fetch。

**Q14：CORS 跨域错误？**

- 开发模式默认仅允许 `localhost` 与 `127.0.0.1` 各端口。
- 生产环境必须配置 `HERMES_ALLOWED_ORIGINS` 环境变量为允许的源（逗号分隔）。
- 未配置时非 localhost 源的跨站带 cookie 访问会被拒绝。

**Q15：Linux/macOS 上如何运行？**

- macOS 首次运行若被 Gatekeeper 拦截，在「系统设置 → 隐私与安全性」中放行，或执行 `xattr -d com.apple.quarantine <可执行文件>`。

**Q16：Android 上如何运行？**

- 通过 `adb push` 将二进制与 `static/` 目录推送到设备（如 `/data/local/tmp/`），赋权后运行。
- 设备需开启 USB 调试。运行后用设备浏览器访问 `http://127.0.0.1:8787`，或用 `adb forward tcp:8787 tcp:8787` 转发到电脑访问。

**Q17：上传文件提示「CSRF token mismatch」？**

- 这是浏览器缓存了旧版前端导致 XHR 上传未携带 CSRF token 头，强制刷新浏览器（`Ctrl+F5`）即可。
- 若仍出现，确认服务端 `webuiVersion` 已升级并重新编译运行。
